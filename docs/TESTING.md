# Test Strategy

- Unit: domain invariants, authorization policies, normalization, scheduling, error mapping.
- Architecture: module/reference boundaries and absence of provider DTO leakage.
- Integration: PostgreSQL through Testcontainers, migrations, outbox, leases, idempotency.
- Provider contract: redacted fixtures across supported versions plus optional live smoke tests.
- API: OpenAPI conformance, RFC 9457 errors, permission matrix, pagination stability.
- Frontend: component states, accessibility, localization parity, generated client contract.
- Accessibility: focused Chromium keyboard journeys and axe-core WCAG 2.2 A/AA rules cover every rendered anonymous/authenticated dashboard state; the audit scope, manual release checks, and limitations are recorded in `docs/ACCESSIBILITY.md`.
- End-to-end: bootstrap, local login, Authentik test realm, onboarding, search dry-run, partial bulk failure.
- Security: SAST, dependency/container scans, secret scan, SSRF cases, auth rate limits.
- Independent automated review: CodeQL and Trivy are complemented by a pinned OWASP ZAP passive baseline against the built read-only image; `docs/THREAT_MODEL.md` distinguishes these gates from the human GA review.
- Credential boundary: strict 32-byte canonical-Base64 key loading, key rotation, randomized AES-GCM round trips, AAD/tamper rejection, encrypted PostgreSQL persistence for independent `api-key`/`username`/`password` purposes, write-only API responses, multi-purpose scheduled resolution, scoped RBAC, CSRF, and redacted audit events.
- Connection boundary: URL normalization, DNS answer-set validation, private/special/NAT64 range rejection, address-pinned redirect-free probes, scoped instance/group CRUD, capability replacement/invalidation, cascade behavior, CSRF, and redacted audit events.
- Arr library contracts: two tagged versions each for Sonarr, Radarr, Lidarr, Readarr, and Whisparr; typed status/health/catalog/queue/history fixtures; series/season/episode/movie, artist/album, and author/book normalization; Whisparr v2/v3 model selection; bounded provider-specific missing-search commands; unknown-field/status/severity tolerance; exact provider/major-version checks; header-only API-key auth; 401/rate metadata; pinned base-path transport; encrypted-key-to-adapter API wiring; and read-only opt-in live smoke tests.
- Supporting Arr contracts: two tagged versions each for Prowlarr and Bazarr, unknown-field tolerance, indexer-configuration exclusion, Prowlarr exact-time history, Bazarr health-object digest redaction, daily subtitle-activity merging, authentication failures, and read-only opt-in smoke tests.
- Download-client contracts: SABnzbd 4.5.3/5.0.4, NZBGet 25.4/26.2, qBittorrent 5.1.4/5.2.3, Transmission 4.0.6/4.1.3, and Deluge 2.1.1/2.2.0 fixtures; exact query/JSON-RPC/form mutation checks; API-key, Basic, login-cookie, and Transmission 409-CSRF authentication; unknown-field tolerance; bounded identifiers; credential-missing outcomes; normalized queue/current-completed history; capability discovery; schedule reconciliation; and read-only opt-in smoke tests.
- Request-manager and ecosystem contracts: Overseerr 1.34/1.35, Jellyseerr 2.7/Seerr 3.3, and Ombi 4.47/4.48 fixtures; exact API-key headers/base paths; status/type/external-ID normalization; title/user/path/reason exclusion; unknown-status tolerance; 10,000-record bounds; health schedule reconciliation; optional read-only live smoke; plus Recyclarr 7/8 version, preview/sync argument, no-shell injection, output-redaction, timeout, and future-version fail-closed coverage through a fake process runner.
- Notification contracts: exact generic HMAC webhook, Discord, Slack, Teams Workflows Adaptive Card, Telegram `sendMessage`, ntfy JSON, Gotify and Pushover payload/auth shapes; success-body and 429 mapping; missing-secret no-call behavior; URL/path/header/body/result redaction; plus implicit-TLS SMTP configuration and redacted request mapping. Live outbound notifications are excluded from the normal suite.
- Media-server contracts: Plex 1.41.9/1.43.2, Jellyfin 10.10.7/10.11.10, and Emby 4.8.11/4.9.3 fixtures; exact token headers and base paths; version/product rejection; item/library and playback/transcode aggregates; discard assertions for users, titles, addresses, devices, and paths; health-only scheduling; unknown-field tolerance; and read-only opt-in smoke tests.
- Web contract: generated-client drift check, strict TypeScript compilation, production Vite build, and dashboard derivation tests proving API counts and permission-absent states do not fall back to demo values.
- Localization contract: exact English/German key and interpolation parity, unsupported-locale fallback, immediate browser switching, timezone-aware `Intl` formatting, and a focused PostgreSQL/API test for preference CSRF, validation, persistence, `/auth/me` projection, and redacted audit.
- Delivery contract: locked dependency restore, warnings-as-errors build, generated-client and migration-model drift, frontend lint/tests/build, non-root read-only image smoke, Trivy repository/image gates, retained CycloneDX SBOM, CodeQL C#/TypeScript analysis, BuildKit provenance/SBOM attestations, and verified keyless Cosign signatures on tag releases.
- Migration command: exact command parsing, empty-schema application of every EF migration under an advisory lock, post-application pending check, idempotent rerun, nonzero failure behavior, valid Compose dependency graph, and a one-shot container exercise against ephemeral PostgreSQL.
- Data lifecycle: isolated PostgreSQL custom dump/list/restore, encrypted credential-row preservation, current migration idempotency, and current-source upgrade from the latest earlier stable image; before the first stable tag, CI uses the initial schema migration as an explicit fallback.
- Scheduler contract: duration/count bounds, timezone-aware Cron calculation, deterministic bounded jitter, capped exponential retry, terminal attempts, handler-to-checkpoint completion, concurrent `SKIP LOCKED` claims, opaque lease-token fencing, retry availability, atomic checkpoint success, migration upgrade normalization, and zero EF model drift.
- Catalog synchronization: natural-key uniqueness, complete-snapshot validation, insert/change/no-op/delete diff behavior, selected provider-data persistence, schedule upsert/disable idempotency, instance cascade behavior, successful checkpoint cursors, and zero EF model drift.
- Missing query: transactional projection transitions/backfill, current RBAC scope intersection, literal title and instance/kind/reason filters, protected filter-bound keyset cursors, 1–200 page bounds, zero-result source freshness, stale threshold, user ownership, case-insensitive saved-view uniqueness, CSRF, and redacted audit summaries.
- Aggregated activity: two-version paged queue/history fixtures, unknown enum tolerance, 10,000-record bounds, 30-second cron materialization, atomic queue replacement/history upsert, 30-day retention, case-insensitive same-instance download correlation, RBAC SQL scoping, freshness, and exact-ID non-disclosure.
- Operation model: actor+route idempotency replay/conflict, 24-hour expiry, concurrent serialization, dry-run persistence, 10,000-target validation, per-target success/failure, partial aggregation, cooperative/pre-start cancellation, actor ownership, CSRF, state constraints, bounded valid result JSON, and audit redaction.
- Search operations: strict selected/instance/group/all scope shapes, authorization intersection, explicit excluded/per-instance preview, current-missing eligibility, empty-target rejection, canonical idempotency hashing, dry-run no-call behavior, 100-ID provider batches, integer-only command payloads, per-instance spacing/Retry-After bounds, cancellation checks, and per-target partial failures.
- Import classification: ordered table-driven exact status/state/event rules, unknown-value no-classification behavior, stable rule/category/severity/guidance keys, English/German key parity, title/message independence, and an invariant that retry support remains false without provider contract evidence.
- Health incidents: deterministic message-independent grouping, related-source preservation, safe remediation URI filtering, five-minute schedule reconciliation, atomic resolve/reopen behavior, source first/last sightings, acknowledgement/snooze persistence, 30-day snooze bound, SQL RBAC scoping, CSRF, and raw-diagnostic audit exclusion.
- Transactional live events: same-commit outbox creation and rollback, redacted allowlisted payloads, leaderless `SKIP LOCKED` publication, actor/current-group filtering, invisible-event cursor advancement, ordered replay, expired/unknown cursor snapshot recovery, bounded retention, runtime SSE/OpenAPI contract, and browser snapshot-before-REST reconnect wiring.
- Audit and diagnostics: global-only authorization, exact filters, protected filter-bound descending pagination including equal timestamps, append-only update rejection, bounded transactional retention with self-audit, strict diagnostics allowlist/pseudonyms, exclusion of names/URLs/addresses/summaries/UUIDs, CSRF, ZIP contract, English/German UI, and live invalidation.
- Performance: the opt-in/scheduled envelope uses 100k catalog items, 20 instances, a 10k-target bulk operation, and a 250-request reconnect burst; thresholds, measurements, hardware scope, and reproduction are in `docs/PERFORMANCE.md`.

Run the focused accessibility gate with `pnpm test:e2e`. It contains two Chromium scenarios and is intentionally separate from the broader component and backend suites.

Coverage is diagnostic, not a target substitute. Changed critical application/domain code should normally exceed 80% branch coverage. Release candidates require a clean install and upgrade from the previous stable version.

Local-identity unit coverage includes strict Argon2id PHC handling, opaque token parsing, generic failures, pre-KDF account/IP throttling, and session lifecycle outcomes. PostgreSQL integration tests exercise the legacy-to-auth migration, concurrent bootstrap synchronization, family-serialized refresh rotation/replay races, logout, audit redaction, throttle windows/lock contention, and migration-aware readiness. API integration tests cover fail-closed bootstrap configuration, bootstrap credential synchronization, CSRF, `__Host-` cookie attributes, fallback authentication, malformed/oversized RFC 9457 errors, rotation/replay, logout, raw login/refresh/logout request caps, and rejection of untrusted forwarded client addresses.

OIDC unit tests cover claim bounds, verified-email policy, exact group mappings, token/session issuance, and logout-context lookup. PostgreSQL tests cover same-subject and same-email races, identity precedence, passwordless provisioning, inactive/missing-role failures, OIDC-only role reconciliation without disturbing manual roles, session-method rotation, protected logout context, and redacted audit. Fast API protocol tests use Authentik-shaped discovery/token/JWKS endpoints with real RSA signatures; they must prove S256 verifier exchange, exact redirects, state/nonce/correlation, issuer/audience/lifetime/signature failures, key rollover, callback rate limiting, secure opaque cookies, refresh, and RP logout. A separate pinned `ghcr.io/goauthentik/server:2026.5.4` suite validates both the provider contract and ArrControl's real handler/session/logout interoperability; it is compiled by the solution, runs only with explicit opt-in variables, and never targets a live deployment.

RBAC unit tests cover the ten-code catalog and system-role matrix, union and global-dominance rules, denial without any applicable grant, ungrouped-instance behavior, validation of custom roles, and manual assignment normalization. Focused PostgreSQL/API tests cover migration seeding, custom-role creation, complete transactional replacement of manual assignments without changing OIDC-owned rows, immediate revocation, OIDC-session isolation, mutation audit, and rejection of a change that would remove the last active global authorization manager.

The focused API test covers runtime OpenAPI paths, `GET /auth/me`, CSRF on role mutation, custom-role reads, immutable system-role rejection, assignment replacement, lockout rejection, and exact RFC 9457 problem codes. Its instance-list matrix proves `401 authentication_required`, `403 access_denied`, global visibility, SQL-filtered group visibility, immediate permission removal, exclusion of ungrouped instances without a global `instances.read` grant, and OIDC grants that do not leak into a local session. These checks use the narrow unit and API/persistence suites; repeating the Authentik browser or container-image suites is not required for RBAC-only changes.
